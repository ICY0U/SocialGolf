// MATERIAL SHOWCASE TROUBLESHOOTING GUIDE
// Fix for "not changing colour + giving debug prompt of interacted + colour"

=== PROBLEM DIAGNOSIS ===

? Interaction is working (you see debug messages)
? Material showcase is visible (orange grid material)
? Material not cycling when pressing E

=== STEP-BY-STEP FIX ===

1. CREATE MATERIAL PRESET DATA ASSET:
   - Content Browser ? Right-click ? Miscellaneous ? Data Asset
   - Choose "SGMaterialPreset" as parent
   - Name: "DA_CustomizableGridPresets"
   - Save location: /Game/SocialGolf/Materials/

2. CONFIGURE THE DATA ASSET:
   - Open DA_CustomizableGridPresets
   - In "UV Presets" array, click + to add entries
   - Add at least 3-5 entries for testing:
     
     Entry 0:
     - Preset Name: "Orange Grid"
     - Material: /Game/CustomizableGrid/Materials/PresetsUV/M_Grid_UV_preset_001
     - Is UV Preset: True
     
     Entry 1:
     - Preset Name: "Blue Grid"
     - Material: /Game/CustomizableGrid/Materials/PresetsUV/M_Grid_UV_preset_002
     - Is UV Preset: True
     
     Entry 2:
     - Preset Name: "Green Grid"
     - Material: /Game/CustomizableGrid/Materials/PresetsUV/M_Grid_UV_preset_003
     - Is UV Preset: True

3. ASSIGN TO SHOWCASE:
   - Find your SGMaterialShowcase actor in the level
   - Select it
   - In Details panel, find "Material Manager" component
   - Set "Material Preset Asset" to your DA_CustomizableGridPresets

4. TEST WITH ENHANCED DEBUGGING:
   - Play the game
   - Walk to showcase and press E
   - Check Output Log for these messages:
     
     ? GOOD: "SGMaterialManager: Cycling UV preset from X to Y"
     ? GOOD: "SGMaterialManager: Set UV preset to X"
     ? GOOD: "SGMaterialManager: Getting UV preset material at index X"
     ? BAD: "SGMaterialManager: No MaterialPresetAsset assigned!"
     ? BAD: "SGMaterialManager: MaterialPresetAsset has 0 UV presets!"

=== COMMON ISSUES & FIXES ===

? "No MaterialPresetAsset assigned!":
   ? Go to showcase actor ? Details ? Material Manager ? Set Material Preset Asset

? "MaterialPresetAsset has 0 UV presets!":
   ? Open your data asset ? Add entries to UV Presets array

? "Getting UV preset material at index X: NULL":
   ? Check material references in data asset ? Ensure paths are correct

? Materials still not changing:
   ? Verify you're using the correct CustomizableGrid material paths
   ? Check that materials exist in /Game/CustomizableGrid/Materials/PresetsUV/

=== QUICK TEST SETUP ===

If you want to test immediately without CustomizableGrid materials:

1. Create data asset as above
2. In UV Presets array, add 3 entries using basic engine materials:
   - Entry 0: Material = /Engine/BasicShapes/BasicShapeMaterial
   - Entry 1: Material = /Engine/EditorMaterials/MatInstEdit_Preview
   - Entry 2: Material = /Engine/MapTemplates/Sky/M_Sky_Panning_Clouds2
3. Assign to showcase and test

=== EXPECTED BEHAVIOR AFTER FIX ===

? Press E ? Material changes visually
? Output Log shows cycling messages
? Different colored/textured materials appear
? Can cycle through all materials in the preset

=== DEBUG CONSOLE COMMANDS ===

Press ` (backtick) and try:
- showdebug input - Shows interaction debug
- stat game - Shows game statistics

=== VERIFICATION CHECKLIST ===

? Data asset created (SGMaterialPreset type)
? UV Presets array has entries with valid materials
? Material Preset Asset assigned to showcase component
? Materials exist at specified paths
? Enhanced debug logging shows proper cycling

If following this guide doesn't work, check the Output Log for the specific
error messages and match them to the fixes above!
